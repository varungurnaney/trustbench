# Information Security Policy

**Document ID:** POL-ISEC-001
**Version:** 1.3
**Effective Date:** June 15, 2024
**Owner:** IT Department
**Classification:** Internal
**Organization:** Meridian Healthcare Solutions

---

## 1. Purpose

This policy defines the information security requirements for Meridian Healthcare Solutions to protect the confidentiality, integrity, and availability of organizational information assets.

## 2. Scope

This policy applies to all employees, contractors, and third-party vendors who access Meridian Healthcare Solutions information systems and data.

## 3. Information Security Principles

Meridian Healthcare Solutions is committed to:

- Protecting patient data and business information from unauthorized access
- Maintaining compliance with HIPAA, SOC 2, and applicable state regulations
- Ensuring business continuity through appropriate security controls
- Fostering a culture of security awareness among all personnel

## 4. Roles and Responsibilities

### 4.1 Chief Information Security Officer (CISO)

The CISO is responsible for:

- Overseeing the information security program
- Reporting security posture to executive leadership
- Approving security policies and standards
- Managing the security budget and resource allocation

### 4.2 IT Security Team

The IT Security team is responsible for:

- Implementing and operating security controls
- Monitoring security events and responding to incidents
- Conducting vulnerability assessments and penetration tests
- Managing access control and identity management

### 4.3 All Employees

All employees are responsible for:

- Completing annual security awareness training
- Reporting suspected security incidents to the IT Help Desk
- Following acceptable use policies for company devices and systems
- Protecting credentials and not sharing passwords

## 5. Security Controls

### 5.1 Access Control

- Role-based access control (RBAC) is implemented across all production systems
- Multi-factor authentication is required for all remote access
- Privileged access requires manager and CISO approval
- Access reviews are conducted quarterly by system owners

### 5.2 Network Security

- Firewalls and intrusion detection systems protect the network perimeter
- Network segmentation isolates production, development, and corporate environments
- VPN is required for remote access to internal resources

### 5.3 Data Protection

- Data classification levels: Public, Internal, Confidential, Restricted
- Encryption at rest using AES-256 for Confidential and Restricted data
- TLS 1.2+ required for all data in transit
- DLP controls monitor outbound data transfers

### 5.4 Incident Response

- Incidents are classified by severity (P1-P4)
- P1 incidents require notification to the CISO within 1 hour
- Post-incident reviews are conducted for all P1 and P2 incidents
- Lessons learned are documented and shared with relevant teams

## 6. Security Awareness

- All employees must complete security awareness training within 30 days of hire
- Annual refresher training is mandatory
- Phishing simulations are conducted monthly
- Training completion rates are reported to HR quarterly

## 7. Compliance

This policy supports compliance with:

- SOC 2 Trust Services Criteria
- HIPAA Security Rule
- State data breach notification laws
- Contractual security obligations with customers

## 8. Enforcement

Violations of this policy may result in disciplinary action up to and including termination. Security incidents resulting from policy violations will be investigated by the CISO and HR.

## 1. Purpose

This policy establishes requirements for governing access to Prism Analytics data assets across production and non-production environments, with specific requirements for data masking when production data is used in non-production contexts.

## 2. Scope

This policy applies to all Prism employees, contractors, and third-party personnel accessing Prism data systems:

- Production databases (PostgreSQL on Amazon RDS)
- Staging and QA databases
- Data warehouses (Snowflake)
- File storage (AWS S3)
- Analytics platforms (Looker, internal dashboards)

## 3. Data Access Authorization

### 3.1 Least Privilege

All data access must follow least-privilege principles. Users receive only the minimum permissions necessary for their specific job function.

### 3.2 RBAC

Access managed through predefined roles. Custom permissions prohibited without CISO exception.

### 3.3 Approval Chain

- Internal data: Manager approval
- Confidential data: Manager + Data Owner
- PII / Restricted data: Manager + Data Owner + Privacy Officer + CISO

## 4. Data Masking Requirements

### 4.1 Non-Production Data Protection

Non-production environments (staging, QA, development) must not contain unmasked production data. The risk of unauthorized PII exposure is elevated in non-production environments due to broader access controls and reduced monitoring.

### 4.2 Masking at Replication

All production data replicated to non-production environments must have PII masking applied at the replication layer before data is written to non-production storage. The masking pipeline must be automated and validated quarterly.

### 4.3 PII Definition

For purposes of this policy, PII includes:

- Full names
- Email addresses
- Phone numbers
- Social security numbers
- Date of birth
- Financial account numbers (credit card, bank account)
- Physical addresses
- IP addresses associated with user accounts

### 4.4 Masking Methods

| PII Field | Required Masking Method |
|-----------|----------------------|
| SSN | Format-preserving encryption |
| Credit card numbers | Tokenization |
| Phone numbers | Partial redaction (last 4 digits preserved) |
| Date of birth | Generalization (year only) |
| Names | Synthetic name replacement |
| Email addresses | Synthetic email generation |
| Physical addresses | City/state only (remove street) |

### 4.5 Masking Validation

IT Security validates masking effectiveness quarterly by sampling 100 records from each masked dataset and confirming no real PII is present.

## 5. Synthetic Data

Where possible, non-production environments should use fully synthetic data generated by the Prism Synthetic Data Generator (SynGen v2.1) rather than masked production data.

## 6. Access Lifecycle

### 6.1 Provisioning

ServiceNow-based access requests with approval chain per Section 3.3.

### 6.2 Contractor and Temporary Accounts

Contractor accounts must have a defined expiration date not exceeding the contract end date. Monthly review by contractor sponsor.

### 6.3 Revocation

Access revoked within 24 hours of termination. Quarterly access reviews for all environments including non-production.

## 7. Exceptions

CISO-approved with compensating controls. Maximum 6 months. Documented in exception register.

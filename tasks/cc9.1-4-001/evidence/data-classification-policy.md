# Data Classification Policy

**Document ID:** POL-DC-001
**Version:** 1.3
**Effective Date:** January 2025

## Classification Levels

| Level | Description | Examples |
|-------|-------------|----------|
| Public | Non-sensitive | Marketing content, blog posts |
| Internal | Business operations | Revenue reports, roadmaps, engineering docs |
| Confidential | Customer PII, business-critical | Names, emails, usage data, contracts |
| Restricted | Sensitive PII, financial | SSN, payment card data, health info |

## Handling Requirements

- Confidential and Restricted data must be encrypted at rest and in transit.
- Access to Restricted data requires CISO approval.
- Third-party processing of Confidential or Restricted data requires a DPA.
